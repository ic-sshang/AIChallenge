Jira Label Mapping
The new labels have been created to do better tracking and metrics reporting. Labels are always the short-term solutions for a long-term problem, until we find a solution labels are the way forward.
Jira LABEL
Description
ESCALATION
The label is used to indicate that this ticket has been escalated to Dev Support Leadership and this have to handled as such with importance.
COMPLEX 
This needs some deep analysis and research. It is quite natural that it would take time hence the WIP
CS_REVIEW
The ticket was send back for CS Review after the initial triage
DS_REVIEW
The ticket had to be reviewed for insights and actions
CONSULT
The Scrum team was consulted for this incident.
TAX_SEASON
Label to mark an item for Tax Season
CRITICAL_JOBS
Label to mark an item for Critical Job Failures that we notice as part of our Daily Critical Job Checks
JA_RESOLVE
Label to mark an item for automation to close the canceled ticket
JA_CANCEL
Label marked to cancel in our end
MI_REQUIRED
Label to mark an item that requires priveleged access to Prod
BROWSERSTACK
Need to use BrowserStack for debug purpose
DS_AUTO
Label to mark an item for automation to auto assign 
CS_ASSIST
Meeting with the Biller to troubleshoot and provide engineering support to CS
What determines the incident to be COMPLEX
The tickets go down several tracks to piece information together and identify the mitigation or/and resolution.
Triage
Database SQL
Application logs - New Relic and Other places
Looking for info from CRM/Biller Portal
Doing code walkthrough of Stored Proc or Application Code
Reading product literature including Guru Cards
Holding consult call with team or engineering team
Additional logging for lack of information
Mitigation
DB ITR
Config changes to Biller
Updating PDF markers etc
Resolution
File a system Bug with root cause details
File a product enhancement with additional details on user/product behavior
An incident can be considered complex if it involves more than three triage tracks and one or more mitigation/resolution steps.
