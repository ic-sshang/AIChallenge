There are 4 types of CAPA issues that can be logged: Quality Issue, Go-Live Slippage, Documentation Request, and Process Improvement Suggestion.  To log a CAPA issue, you’ll need to create a new ticket in the CAPA Jira project located here: 
Create New CAPA Issue
Steps:
Click the above link to create a new issue
Select issue type:
Quality Issue
 
- Select quality issue for any quality issue that either impacted the customer’s experience or could have impacted a customer.  Log the following fields then click Create button:
Summary 
- High level summary of the issue.
Requesting Billers
 - Biller project that the quality issue occurred.
Description 
- Detailed description of the issue.
Priority 
- Select the appropriate priority level
Highest 
- Severe issue that significantly impacted the customer.  Caused major delays or pushed go lives and resulted in rework by us or the biller.
High 
- Severe issue that impacted the customer.  Resulted in some delays or minor go live change.
Medium 
- Did not significantly impact the customer but is an issue that needs to be addressed.  Caused minor delays and possibly some rework.
Low 
- Not customer impacting but process inefficiency that needs to be addressed.
Lowest 
- Do not use 
Go-Live Slippage
 
- Log a GL Slippage ticket if a Go Live date for an active implementation pushes 
out of the current 
GL quarter
.  Log the following fields then click Create button:
Summary
 - Summary of which biller GL slipped
Requesting Billers
 - Biller whose GL slipped.
Description
 - Detailed description of whey the GL date slipped.
Current GL Date
 - Date biller was scheduled to go live.
New GL Date 
- New GL date.
Documentation Request
 - Submit a request to have new documentation created for a process, tool, or topic.  Log the following fields then click Create button:
Summary
 - Summary of documentation requested
Description
 - Detailed description of what type of documentation you’re requesting.
Label
 - Documentation
Process Improvement Suggestion 
- Submit a suggestion to improve an existing process or create a new one.
Summary
 - Summary of process improvement suggestion
Description
 - Detailed description of process improvement.  Include expected benefit as well.
Track the status of submitted requests here: 
CAPA Issues in progress
  or go to the Kanban board located here: 
https://invoicecloud.atlassian.net/jira/software/c/projects/CAPA/boards/281
