Definition
A Jira status that indicates that the item needs to be reviewed or is currently being reviewed by one or more peers or tech leads. The review will consist of making sure that the design and logic of the code match the deliverables on the Jira item description. 
Process: 
Peer Review:
Move your item from In Progress to Tech Lead review.
Request a reviewer in the #idev-support Slack channel (or via direct message).
Add the reviewers as “Optional Reviewers” on the item’s associated Pull Request.
Reviewers will review the design and logic of the code and add in-line comments on the PR as needed.
The developer will address comments by making changes and Resolving the comments. 
If changes are made on the PR, the tester must retest the item and the Reviewer must review the PR again. 
Once all changes have been made and the PR has been approved, the Reviewer should move the item to CRC Review.
