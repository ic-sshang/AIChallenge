Purpose
The purpose of 
bug court 
is to review, prioritize, and allocate bugs to the appropriate team for resolution.
Agenda Items
Meeting is set to officially start at 1:02 PM Central Time. (Tuesday & Thursday 
Only
)
Bug Review
A Product Reliability Engineer will present the bug items to the product team.
(Optional) The initial reporter (CS) then could add additional context to how the issue is affecting the biller(s) and/or software partner(s).
The Product engineer/representative has an opportunity to ask questions or clarify any concerns about the bug item.
The Response Commitment field will be set to 1, 2, or 3 during the meeting. See 
Definitions and Actions
 for the definition of the Response Commitment values.
