Overview
Skybot is a control plane is used to run jobs on agents (App Servers).
Deployment Checks
When deploying a Skybot job the deployment pipeline should:
Run the job’s Health Checks on the target deployment App Server, there can be more than one App Server, by executing the job via command line and passing the “-healthcheck” argument.
Trigger the job using the Skybot API after all of the target deployments to App Servers have completed successfully.  The trigger should pass the “healthcheck” parameter to the Skybot API which is added to only that trigger’s ENVIRONEMNT VARAIBLEs run.
