Highlights
Development sprints will occur every 2 weeks
All User-facing changes (including CRM) will be rolled up into a single Monthly Release
Weekly Release Window is restricted to critical bugs and non-user facing changes
Monthly Pre-Release will be made available to internal and Beta users one week before the Monthly Release
Monthly Pre-Release will be accompanied by Internal Training
External Communication will happen one day before the Monthly Release
Changes to Production occur Tuesday night, so users will see changes start of business Wednesday
