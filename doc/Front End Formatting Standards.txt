Indentation
To maintain consistency with the existing codebase, use 4 spaces to indent (per indentation level)
Avoid using tabs
Indentation Examples
Recommended
html
wide
<asp:PlaceHolder ID="phInvoiceTypes" runat="server" Visible="false">
    <div class="col-sm-6">
        <div class="form-group">
            ...
        </div>
    </div>
</asp:PlaceHolder>
Recommended
css
wide
@media (min-width: 768px) {
    .row.display-flex {
        display: flex;
        flex-wrap: wrap;
    }
}
Avoid
Offset an nonuniform indentation patterns
wide
<div class="hidden-xs">
    <a href="#paymentSummary" role="button" data-toggle="collapse">Click to view Payment Summary</a>
    <div id="paymentSummary" class="collapse">
                <IC:PaymentSummary ID="PaymentSummary1" runat="server" />
    </div>
</div>
Avoid
Double space indentation -- more difficult to read
wide
<asp:PlaceHolder ID="phInvoiceTypes" runat="server" Visible="false">
  <div class="col-sm-6">
    <div class="form-group">
      ...
    </div>
  </div>
</asp:PlaceHolder>
Blank Lines
For readability, add blank lines to separate large or logical code blocks.
Avoid adding more than one consecutive blank line
Keep line breaks consistent
Always leave one line at the end of a file
Blank Lines Examples
Recommended
html
wide
<div class="table">
    <div class="table-cell">
        <h1 class="checkout-currentstep sso-checkout-currentstep">Payment Options</h1>
    </div>

    <div class="table-cell">
        <span class="checkout-step sso-checkout-step">Payment Information</span>
    </div>

    <div class="table-cell">
        <span class="checkout-step sso-checkout-step">Review Payment</span>
    </div>
</div>

Avoid
Inconsistent line breaks between code blocks
html
wide
<div class="table">
    <div class="table-cell">
        <h1 class="checkout-currentstep sso-checkout-currentstep">Payment Options</h1>
    </div>


    <div class="table-cell">
        <span class="checkout-step sso-checkout-step">Payment Information</span>
    </div>

    <div class="table-cell">
        <span class="checkout-step sso-checkout-step">Review Payment</span>
        
    </div>
</div>
Line-Wrapping
For readability, wrap longer lines (when there are more than 4 attributes or the line is too long visually)
Indent each successive line 
Alphabetize the order of attributes
The closing bracket > should then be alone on it’s own line
Line-Wrapping Examples
Recommended
html
wide
<div>
    <asp:DropDownList
        AutoPostBack="false"
        CssClass="form-control"
        ID="ddlPaymentType"
        runat="server"
        Width="248px"
    ></asp:DropDownList>
</div>
Avoid
html
wide
<span id="GooglePayButtonSpan" role="tooltip" class='GooglePayButtonHelper' data-toggle="tooltip" data-html="true" data-placement="right" title='<p class="GooglePayButtonHelperP">After clicking the "Google Pay" button, you will be directed to Google Pay to choose a payment method and will return to the site to finalize your payment.</p>'>What is this?</span>
Capitalization
HTML element names, attributes, attribute values, CSS selectors, properties, and property values should all be lowercase
Capitalization Examples
Recommended
html
<img src="root/mypic.png" alt="portrait of a man">
css
color: #e5e5e5;
Avoid
html
<IMG SRC="root/mypic.png" ALT="portrait of a man">
css
color: #E5E5E5;
Comments
Remove unnecessary comments.
Don’t use comments to save legacy markup. If we need to look at what used to be there we can look at commit history.
Code should be self documenting whenever possible, but comments can be used to explain code
Comments Examples
Avoid
css
wide
/*new*/

.sidebar-sub-title {
    font-size: 12px;
}
css
wide
.product {
    background-color: #24B6DC;
    padding: 11px 11px 11px 16px;
    /*border-radius: 36px;*/
    text-align: center;
    font-size: 20px;
    color: #fff;
    width:55px;
    display:inline;
}
c#
wide
<%--<asp:HyperLink ID="lnkOneTimePay" runat="server"></asp:HyperLink>--%>

Class Naming
Avoid using cryptic names
Avoid using presentational names
Strive for meaningful class names that speak to the purpose of the element being targeted
Use a hyphen as a delimiter to separate words
Class Naming Examples
Recommended
.gallery {}
.login {}
.video-id {}
Avoid
.demoimg {}
.atrx {}
.yas-5667 {}
.lgn {}
.button-green {}
Property Order
Whenever cascade is not involved, alphabetize class properties
Property Order Examples
Recommended
.button {
    border: 2px solid black;
    color: #fff;
    margin: 3px;
}
Avoid
.button {
    margin: 3px;
    color: #fff;
    border: 2px solid black;
}
Shorthand Properties
Whenever possible, use shorthand properties
Shorthand Property Examples
Recommended
css
padding: 0 0 10px 12px;
Avoid
css
padding-top: 0;
padding-right: 0;
padding-bottom: 10px;
padding-left: 12px;
!Important Declarations
Avoid using !important wherever possible
These declarations break the natural cascade and make it challenging to reason about create styles
!important Examples
Recommended
css
.button,
.custom-button {
    background-color: #fff;
    color: #000;
}

.custom-button {
    color: red;
}
Avoid
css
.button,
.custom-button {
    background-color: #fff;
    color: !important #000;
}

.custom-button {
    color: red;
}
Property & Class Name Stops
Always include a single space between the property and value
Always include a single space between the last selector and the declaration block
Property & Class Name Stop Examples
Recommended
css
.full-account-balance-amount {
    padding-left: 20px;
    text-decoration: underline;
}
Avoid
css
.full-account-balance-amount{
    padding-left:20px;
    text-decoration:underline;
}
Rule Separation
Separate CSS rules with new lines
Rule Separation Examples
Recommended
css
.summary-table th {
    border-bottom: solid 1px #f1f1f1;
    color: #666;
    padding: 5px;
}

.summary-table td {
    padding: 5px;
}
Avoid
css
.summary-table th {
    color: #666;
    border-bottom: solid 1px #f1f1f1;
    padding: 5px;
}
.summary-table td {
    padding: 5px;
}
Declaration Block Separation
The opening brace should be on the same line as the last selector 
Declaration Block Separation Examples
Recommended
css
.summary-table th {
    border-bottom: solid 1px #f1f1f1;
    padding: 5px;
}
Avoid
css
.summary-table th
{
    border-bottom: solid 1px #f1f1f1;
    padding: 5px;
}
Selector and Declaration Separation
Separate selectors and declarations by new lines
Always start a new line for leach selector and declaration
Selector and Declaration Examples
Recommended
css
a:focus,
a:active,
link,
link:focus {
    color: #666;
}
Avoid
css
a:focus, a:active, link, link:focus {
    color: #666;
}
Semantics
Use elements for what they’ve been created for
Avoid using a ton of <div> elements wherever possible
Semantic Examples
<p> for paragraph, 
not
 another <div>
<a> for links 
not
 <div onclick=”someFunction()”>Link</div>
