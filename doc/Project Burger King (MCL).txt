The supported model is a 
multi-tenant, role-based access system 
with the ability for 
linked accounts.
A User has one Login*, and that Login can be linked to multiple Customer Accounts.

*A user has one Login but there are scenarios where many Logins are created with the same email address. This is a documented issue.

Problem Patterns Identified from Support Cases
Role Removal Limitations
Owners and co-owners cannot remove one another, leading to stalled cases and manual intervention.
This becomes problematic during policy changes, account transfers, or user disputes.
Automatic Role Changes Creating Confusion
When an owner is unregistered, the oldest co-owner is automatically promoted to owner without explicit admin control or notification ‚Äî leading to unintended authority shifts.
Bulk Ownership Updates via Spreadsheets
Many cases request mass uploads or removals from lists, which is error-prone and time-sensitive.
Timing differences between reports cause misalignment between system state and intended changes.
Dependency on Email as the Sole Identifier
Multiple cases show reliance on email address matching to determine account ownership ‚Äî causing errors when emails change or are reused.
