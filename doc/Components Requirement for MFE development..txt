Components:
Header
Support for multiple heading levels: 
H1 through H6
, allowing for structured and hierarchical content display.
Autopay Icon
A visual icon representing the 
Autopay
 feature for quick recognition and interaction.
Typography
Consistent and customizable 
text styles
 for headings, body text, captions, and more.
Grid/
Table
Sortable columns
: Users can sort data in 
ascending or descending
 order.
Each cell can render 
icons, buttons, or icon-text combinations
 for enhanced interactivity.
Grid component should support Pagination. 
Select Dropdown
A dropdown component allowing users to 
choose from a list of options
.
Radio Button
A group of 
mutually exclusive options
, where only one can be selected at a time.
Button
Buttons should support 
Primary
 and 
Secondary
 styles for different levels of emphasis.
Account Links
A component for displaying 
navigational links
 related to user account actions or settings.
Checkbox
A toggleable box that allows users to 
select one or more options
 independently.
Phone Number Input Box
An input field specifically designed for 
phone number entry
, with formatting and validation.
Alert
Alerts can be of various types: 
Success, Warning, Error, Info
, etc., to convey different levels of urgency or feedback.
PayByText Icon
An icon representing the 
Pay by Text
 feature for visual cues in UI.
Paperless Icon
A symbol indicating 
paperless billing or communication
 preferences.
Filter
A component to 
refine or narrow down
 data based on selected criteria.
Info Icon
An icon that provides 
additional context or help
 when hovered or clicked.
Loader / Skeleton
Visual indicators like 
spinners or skeleton screens
 to show loading states and improve perceived performance.
