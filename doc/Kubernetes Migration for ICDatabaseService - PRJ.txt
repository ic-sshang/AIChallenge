Notes
Query Structure & Formatting
Use consistent casing for NRQL keywords (SELECT, FROM, WHERE, etc.). Indent and format queries for readability. Remove unnecessary whitespace.
Faceting & Filtering
Avoid faceting on ephemeral or host-based fields (e.g., Machine, HostName). Update queries to facet on stable, logical identifiers (e.g., serviceName, namespace, clusterName, or custom K8s labels). Replace hardcoded hostnames or patterns with dynamic tags or labels. Avoid brittle filters like NOT IN (...) or LIKE '%Temp%'.
Time Windows & Timeseries
Use relative time windows (SINCE 1 day ago, SINCE 1 week ago). Ensure TIMESERIES intervals are appropriate. Avoid hardcoded or overly broad time windows unless justified.
Query Efficiency
Avoid overuse of LIMIT MAX; use a reasonable LIMIT value unless all results are truly needed. Limit the number of facets to what is actionable and useful. Test queries in the New Relic Query Builder for performance.
Custom Attributes & Instrumentation
Ensure all custom attributes used in queries (e.g., serviceName, environment, MethodName) are consistently set and available in K8s. Update instrumentation as needed to provide new or renamed attributes after migration.
Error Handling & Data Quality
Ensure all fields used in queries are present in all relevant events. Use IS NOT NULL or default values to avoid missing data in dashboards.
Documentation & Ownership
Document any changes to attribute names, values, or query logic as part of the migration. Assign ownership for each dashboard and alert to ensure ongoing maintenance.
Testing & Validation
After migration, validate that all dashboards and widgets display expected data. Compare pre- and post-migration results for key metrics. Solicit feedback from dashboard users and stakeholders.
