Risk Analysis
If the boarding API is down, we will not be able to pass information back and forth to Chase. This will create a delay in onboarding billers or making updates to their accounts. Is there a back up process that we could use should this occur? 
If the APIs are not functioning correctly at the time of roll out, we need to establish a plan to continue business as usual until the issue is resolved. 
Need to build out all risk scenarios to minimize any downtime as this change is implemented. 
We would require a roll back plan should there be significant issues at the time of roll out.  
3 weeks
