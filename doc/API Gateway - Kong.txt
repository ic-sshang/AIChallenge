Overview
Kong API Gateway is a web traffic load balancer that operates at the application layer (OSI layer 7). It is used to route API HTTP traffic to our Azure Web Servers and k8s cluster ingresses.  It is also used to manage API lifecycles, centralize authentication, and observability.
Each target persona (API user) and environment will have a workspace in Kong.  Workspaces isolate Kong resources;  services, routes, plugins, and consumers.
