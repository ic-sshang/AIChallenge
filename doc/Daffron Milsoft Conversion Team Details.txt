Summary
This document outlines the technical and procedural steps for converting a biller’s software platform to Daffron/Milsoft. The process includes handling of autopay and paperless enrollments, customer re-registration, account number mapping, and CRM configuration updates. The goal is to preserve customer payment continuity while transitioning to the new system.
 Process
Definitions
Autopay
: Automatic recurring payment enrollment.
BID
: Biller ID used in InvoiceCloud to identify billing systems.
CRM
: Customer Relationship Management system used to manage biller configurations.
Daffron/Milsoft
: Integration partner responsible for importing data into Daffron’s eBusiness system.
Paperless
: Customer enrollment in electronic bill delivery.
SFTP
: Secure File Transfer Protocol used for file exchange.
Token Registration
: A secure method for registering users without transferring passwords.
Key Responsibilities
Biller
:
Confirms go-live date.
Notify customers about re-registration.
Provide Daffron with account mapping file.
Project Manager (InvoiceCloud)
:
Coordinate with Daffron and Milsoft.
Prepare and deliver autopay and paperless files.
Oversee CRM updates and system maintenance.
Ensure secure handling of customer data.
Provider (Daffron / Milsoft)
:
Provide new account numbers.
Import customer data into Daffron.
Configure EMV readers and CloudPayV2.
Return cross-reference file for account mapping.
Quality Assurance
Project Manager
:
Validate that autopay and paperless flags are preserved.
Confirm CRM settings are updated correctly.
Ensure customer portal authentication is functioning post-conversion.
Biller
:
Confirm customer access and payment functionality post go-live.
Roles
Biller
: The client organization transitioning to Daffron.
Project Manager (InvoiceCloud)
: Oversees the conversion process and coordinates data handling.
Provider
: Daffron (CIS) and Milsoft (integration partner).
Procedure
Initial Coordination
Who:
 Project Manager
Confirm go-live date and 3-day maintenance window.
Notify biller that customers will need to re-register.
Coordinate with Daffron regarding test BID.
Prepare production BID 2806.
Autopay & Paperless Data Handling
Who:
 Project Manager
Prepare file with:
Customer name
UTI account number
Email
Paperless flag (Y/N)
Autopay flag (Y/N)
Default payment method type (card/check)
Last 4 of payment method
Send to Daffron for system update.
Receive cross-reference file with new Daffron account numbers.
Use this to update account numbers in IC during maintenance window.
Customer Registration & Authentication
Who:
 Project Manager
Do not transfer passwords; customers must re-register.
Use token-based registration if needed.
Update CRM:
Profile > Misc Settings → Username with Password (via DBR ticket)
System Configuration & CRM Updates
Who:
 Project Manager
Reconfigure Production BID using setup from Test BID .
Unregister payers from Production BID during maintenance.
Ensure autopay and paperless remain intact post-update.
Confirm that customers can manage payment methods post go-live.
Milsoft Integration & Technical Setup
Who:
 Provider (Milsoft), Project Manager
Milsoft to import data into eBusiness.
Configure EMV readers for Daffron’s Cash Drawer.
Milsoft to code to CloudPay
V2
.
Request two Data Pump threads (reference Biller or CDE Lightband).
Include this in the Platform Conversion request.
Post-Go Live Validation
Who:
 Project Manager, Biller
Confirm that autopay and paperless settings are preserved.
Validate customer access and payment functionality.
Ensure CRM reflects updated account numbers and authentication settings.
