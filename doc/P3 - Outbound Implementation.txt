SMART Exchange/Disperse API Credentials
Receive Postman collection from Transcard and send to biller and core system
Transcard will send you the biller specific Postman collection including their API Credentials
Forward this to the biller and point of contact from their Core System in an 
ENCRYTED
 email including the below language.
The attached Postman collection contains the API credentials, which can be found in the "Variables" section. You will be able to use the collection to run test transactions into the SMART [Disburse/Exchange] portal. There are a few configurable items where you can elect to include a document with the payments, as well as, request an e-signature. Items set to "false", will disable the function.
Please note: The payable (token) delivery method is set to the email variable and will pull from the main collection’s variable. This should be updated to the tester’s email address.
