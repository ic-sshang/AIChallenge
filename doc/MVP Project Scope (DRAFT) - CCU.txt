This document will be used to capture the MVP requirements for the project to meet the existing contract and its deadline.   Additional enhancements can be made after delivering MVP.
Terms:
CIS = Customer Information System
CRM = Customer Relationship Management
Cayenta = Third party software vendor
System
Story
CRM
As a IC employee, I want to be able to set the “Contact Us” ticketing provider, so I can support Billers with different CIS systems.
**There is currently only one provider “Cayenta CIS”
CRM
As a IC employee, I want to be able to manage the “Contact Us” feature for a Biller who uses a supported ticket CIS system, so that I can better serve Billers.
CRM
As a IC employee, I want to configure the URL and API credentials for a Cayenta CIS ticketing system, so I can provide the flexibility for a Biller’s Cayenta CIS installation.
1.5 Payer Portal
As a Payer, I want to login into the Payer Portal and submit a support ticket, so that my Biller can manage my ticket in their CIS system.
1.5 Payer Portal
As a Payer, I want create a Cayenta support ticket with my name, phone number, email, and issue description, so that my Biller can easily assist me.
1.5 Payer Portal
As a Payer, I want to see the Cayenta CIS ticket number after it is created, so that I can record it for reference later.
1.5 Payer Portal
As a Payer, I want to see an error message if a Cayenta ticket can not be submitted, so that I can retry or use a different method to get support.
