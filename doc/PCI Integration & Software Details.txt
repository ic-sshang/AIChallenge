Invoice
CSV
SFTP
The purpose of this file is transfer bill/invoice data to IC from the biller. PCI billers provide IC a ‘full file’ on a daily basis that includes all accounts in their system. This file is used to split into the following: new invoices, balance adjustments, customer updates (block ACH and block CC) and conveyance.
Adjustments/Balances
BAF
SFTP
IC creates the adjustment file from the ‘full file’ the biller provides. The purpose of this file is to update invoice balance data to IC from the biller.
Payments
TXT
SFTP
The purpose of this file is transfer payment data from IC to the biller.
Customer Update*
CUF
SFTP
Integration V1 does not support the Customer Update File
Integration V2 - IC creates the customer update file from the ‘full file’ the biller provides. Based on changes to ‘account status’ in the full file (Custom48), IC will create a customer update file to make updates to block ACH and block CC indicators.
Conveyance*
CCF
SFTP
Integration V1 does not support the Customer Conveyance File
Integration V2 - IC creates the customer conveyance file from the ‘full file’ the biller provides. IC conveys accounts and invoices when the name on the account changes in the daily file for Real Estate invoices only.
