Proposed Workflow:
The following workflow is being attempted to generate a health check code using OpenAI,
Preprocess the noncustom components and come up with a list that chatGPT will continue trying to make a health check against
	Example:
		Component: ICFramework.Utilities.Email
		Files: Program.cs, Email.cs
1. Pass Program.cs and ask AI to generate a health check against it 
2. Pass Email.cs AND previous output. Ask AI to fix/improve previous output with the current context.
3. Once all the files have been analyzed move onto the next component.  
Pros: We can circumvent token limits by slowly giving full context file by file. We end up having more context vs doing it against one file. 
Cons: This will take too many tokens ($$$) and too much time. We will have to repeat the process for all noncustom implementations.
