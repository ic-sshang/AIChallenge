Required Options
ID
Option Description
Details
66
Enable Stored Payment Method via IVR
If enabled, and the IVR partner implements Stored Value/Payment Methods, enabling this option will allow that additional functionality to work. By default, it is disabled for security reasons
Always turn this ON
73
Customer Portal - Allow AutoPay registration for non-registered Customers
If enabled, we will allow a Customer to register for AutoPay even if they are not registered.
Always turn this ON
101
Customer Portal - Require email address when making a payment
By default when making a payment in the Customer Portal, email address entry is not required. By checking this option the user will be required to enter their email address when making a payment.
Always turn this ON
129
Display latest $0 or negative balances in Open Invoices and CloudPayV2
When enabled, the most recent $0 or negative balances will be displayed in the Open Invoices and CloudPayV2. Note: Invoice Type #15 must be enabled for this feature to work.
Always turn this ON
135
Enable CloudPay v2 Single Invoice Postback URL
If enabled, the Postback URL will only contain a single invoice even if multiple invoices are paid. This is to prevent original implementations from having to reupdate their Postback support after we added multiple invoice support to the Postback URL.
Always turn this ON
