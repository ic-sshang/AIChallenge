Summary
The purpose of this page is to give instructions on how to setup the Caselle Cloudstores so that the payment files export properly. There are many different combinations of Distribution Code and Payment Code that are supported in a single file and all configuration is done within CRM.
Sample Biller: Caselle Batch Integration BID 2077
Cloud Store Types
All of the settings below need to be configured in a specific way so that the cloud store distribution code populates properly in the payment file. 
See the differences in the configuration types below.
Single Code
Summary: Cloud Store setup to accept one type of payment with one distribution code. There are no additional variables or fixed pricing variables that need to be considered. 
Example: Library Cloud Store setup with no payment dropdown options or fixed amounts
One-Code for each Dropdown Value
Summary: Payment is being made on a Cloud Store that offers more than one payment type. Each payment type has a different distribution code to be used in the payment file.
Example: Police Department Cloud Store setup with Payment Types for Fingerprinting, Parking, etc.
One-Code for each ‘Item’ Selection
Summary: Payment is being made on a Cloud Store that offers more than one Payment Type that is a Fixed Item. Each payment type has a different distribution code to be used in the payment file.
Example: Maintenance Services Cloud Store setup with Payment Type Fixed Items for Grading Drainage ($250.00 per unit), Park Rental ($20.00 per unit), Yard Waste Ties ($1.50 per unit), etc.
