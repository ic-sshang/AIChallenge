Agent Server Build
Install Windows updates
Turn off Windows firewall
#### Install Crowdstrike
Shutdown sever and stop in Azure portal.
Go onto Azure portal (future build in code) and expand hard drive from 16 gig to 64 gig
Login to server and right mouse click on windows logo and choose Disk Management
On the first unallowcated disk, right mouse click and choose New Simple Volume
Click Next, Next, and make sure the drive letter is set to E:
Install the agent software:
Use File Explorer to create new Temp folder on the E: drive
Copy the setupAutomateScheduleAgent.exe from the E:\Data\Temp\Automate_Schedule_Agent_for_Windows folder on the IC-App8 server to the E:\Temp on the new App server
Run the install
Take the defaults except for Remote server to connect, enter your skybot server here
Click install at the end
Click finish
Enter Yes to start the agent
Login to Skybot server and admin portal and click on Agents tab and verify the new agent shows up
 
ICSkybotService Installation
Reference Azure DevOps Repo: 
https://dev.azure.com/invoicecloud/Src/_git/ICSkybotService
Download the Installer: 
https://dev.azure.com/invoicecloud/Src/_git/ICSkybotService?path=/IC.WindowsService.SkybotService/setup_ic_skybot_service_x86.exe
Install it on the server.
Add a hosts file entry for ICSERVICESSkybot and set to localhost.
Follow this article: 
https://community.helpsystems.com/forums/skybot/general-questions/7872f3b7-1887-e511-80cf-0050568460e4
 
Open up PgAdmin from the following directory :
C:\Program Files (x86)\Skybot\Skybot Scheduler\pgsql\bin\pgadmin3.exe
Create a new user using the SQL found in this file: 
https://dev.azure.com/invoicecloud/Src/_git/ICSkybotService?path=/PostGreSQL/CreateUserScript.sql
Execute the unit tests from this repository: 
https://dev.azure.com/invoicecloud/Src/_git/IC.ServiceProxies.Skybot
