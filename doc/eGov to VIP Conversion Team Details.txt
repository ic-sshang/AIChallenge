Summary
This document outlines the technical and procedural steps required to convert a biller from the SSI eGov platform (batch integration) to the SSI VIP platform (real-time web services). It ensures continuity of service, data integrity, and a seamless transition for both billers and customers, while preserving historical data and minimizing service disruption.

Definitions
Account Mask: A format used to identify account numbers for payment processing.
AutoPay: A feature allowing automatic recurring payments.
BID: Biller ID used to identify the biller in InvoiceCloud.
Biller GUID: Unique identifier for the biller in the system.
eGov: Legacy batch-style integration platform.
Initialization File: File that updates balances in VIP post-conversion.
ITR: Internal Ticket Request for data services.
JIRA: Project tracking tool used for submitting integration and development tasks.
OBD (Online Bank Direct): Payment method involving direct bank transactions.
PDF Extraction: Method of extracting invoice data from PDF files.
SSI VIP: Real-time web services platform replacing eGov.
VIP Template: Invoice format used in the VIP platform.
