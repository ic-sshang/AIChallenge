Summary
Our Sauce Connect Proxy tunnel uses an Allow List approach to internet connectivity. In theory, this is what all of our servers should use, as it’s the most secure approach - the server can only reach out to resources that it needs to function. The reason that all of our servers do not use this approach is that it can be cumbersome to determine what external resources an app needs.
Identifying Domain Names Needed to be Allowed
This is an exercise that we need to perform in the event that an internal app is not running as expected when accessed via Sauce Labs. 
In the Sauce Labs portal, start a Live Test with any browser you’d like. 
Open up the dev tools in that browser and run through the troublesome workflow. 
As you do this, look for any network requests with a status code of 
502
. These ones are getting blocked by our firewall and will need to be added to the allow list. Once you go through a workflow and capture all instances of these blocked requests, you can
 submit a service desk ticket to allow them
.
Please note that 
not all blocked requests should be allowed
. This is a good example: our regression site is attempting to pull resources from our production site. The regression site should not have any communication with production; this should be changed within the app, not allowed by network policy here.
