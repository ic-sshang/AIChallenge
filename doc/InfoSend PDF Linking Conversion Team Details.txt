Summary
This document outlines the key considerations and resources for implementing InfoSend PDF linking. It includes references to Guru documentation, integration steps, and coordination with the print vendor. The process ensures that customer invoices are accessible via remote PDF links and that data dumps are properly configured.
 Process
Definitions
Data Dump: A structured export of billing or invoice data used to link PDFs.
Guru: Internal knowledge base containing implementation guides and documentation.
InfoSend: A print vendor that provides PDF hosting and linking services.
PDF Remote Link: A URL-based method to access hosted PDF documents.
Key Responsibilities
Biller: Provide accurate data dump files and validate PDF links.
Project Manager (InvoiceCloud): Coordinate implementation with InfoSend and ensure documentation is followed.
Provider (InfoSend): Host PDFs and provide remote link access per integration requirements.
Quality Assurance
Biller: Confirm that customer-facing documents are accurate and accessible.
Project Manager: Verify that all PDF links resolve correctly and match the data dump records.
Roles
Biller: The organization providing billing data and requiring PDF linking.
Project Manager (InvoiceCloud): Oversees the implementation and ensures all steps are completed.
Provider: InfoSend, the print vendor responsible for PDF hosting and delivery.
Procedure
Review Guru Documentation
Who: Project Manager
Access and review the following Guru links to understand the InfoSend PDF linking process:
PDF Remote Link Guide - InfoSend (Print Vendor)
Infosend Implementation (Print Vendor)
Coordinate Data Dump
Who: Biller
Prepare and submit the required data dump file to InfoSend. Ensure that all fields necessary for PDF linking are included.
Validate PDF Links
Who: Biller, Project Manager
After InfoSend provides the remote links, validate that each link correctly resolves to the appropriate PDF. Check for accuracy and accessibility.
