Required Options
ID
Option Description
Details
1
Allow Adjustments on Closed Invoices
Enable this option if Payments and Adjustments can be applied to an Invoice that is considered closed. There are cases in which the recommended functionality is to disable this option. One case is Balance Forward, which by definition the balance of the previous Invoice is adjusted out and moved to the new Invoice. No more payments or adjustments should be allowed on that closed Invoice.
Always turn this on
2
Paperless Checkbox Default
When Registering, or when making a Payment via ExpressPay, the user is presented with the option to Go Paperless with a checkbox. The Biller may choose to have this either checked by default, or not checked.
Determine with the biller
64
RTDR - Override 6 Hour Previous RTDR Check
For Billers with RTDR, we normally would not pull a RTDR Balance if we had a successful balance within the past 6 hours. Enable this to Force a refresh.
 
Always turn this on
93 
Virtual Site - Display registration areas
By default, registration panels and menu links may be shown in the virtual site if the Biller allows registration for their invoice types. Turning this off overrides all other checks and hides these areas. Customers may still access the registration page via the invoice search if that invoice type allows registration.
This option is 
On
 by default.
Always turn this off
41
Block Customer Portal from Email Updates/Deletes
By default, the Customer Portal allows a customer to update and remove an email address. This will block that capability. (MLS)
This option is 
Off
 by default.
Always turn this on
42
Biller Portal - Edit Email Address
By default, changes to the email address are allowed. Unchecking this will make the Email Address field in the Biller Portal as read-only.
This option is 
On
 by default.
Always turn this OFF 
101
Customer Portal require email when making a payment
By default when making a payment in the Customer Portal, email address entry is not required. By checking this option the user will be required to enter their email address when making a payment.
Always turn this on
108
Disable Email Address Update from BIF
If enabled, invoice file (BIF) imports will not update a customer email address. However, new emails for customers without an email address, will still be inserted.
This option is 
Off
 by default.
Always turn this on
129
Display latest $0 balance invoices in Open Bill Listing and CloudPayV2
When enabled, the most recent $0 balance invoice will be displayed in the Open Bill Listing and CloudPayV2. Note: Invoice Type #15 must be enabled for this feature to work.
This option is Off by default
Always turn this on
