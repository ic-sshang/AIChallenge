Summary
This document outlines the process for upgrading a biller from enQuesta v4 to v6. The upgrade involves changes to data pump handling, endpoint configuration, and advanced settings. It requires close coordination with S&S and the biller to ensure a smooth transition, particularly for multi-user environments and PayConnect functionality.
 Process
Definitions
Advanced Settings 24–27: CRM settings required for PayConnect integration.
Capricorn: Customer-facing portal used by enQuesta billers.
Data Pump: Service for transmitting payment, paperless, and autopay data.
PayConnect: S&S’s real-time integration service for account sync and payments.
RTDR: Real-Time Data Retrieval, used to fetch account and billing data.
S&S (Systems & Software): Provider of the enQuesta CIS platform.
Token1/Token2: Authentication tokens used in multi-user login environments.
Key Responsibilities
Biller
:
Coordinate with S&S to provide required endpoints and tender types.
Participate in testing during the conversion weekend.
Project Manager (InvoiceCloud)
:
Confirm and schedule the maintenance window.
Request technical contacts and configuration details from S&S.
Submit ITG for test portal setup.
Ensure advanced settings are configured correctly.
Monitor and support testing during conversion weekend.
Provider (S&S)
:
Provide endpoint URLs and payment codes “tender types”.
Configure PayConnect and DNS records.
Perform system-side configuration changes during the upgrade.
Quality Assurance
Project Manager
:
Confirm that RTDR and data pump services are functioning post-upgrade.
Validate that token-based login works for multi-user billers.
Ensure no custom code is missing from the test portal.
If S&S does not receive Token1/Token2, verify Advanced Settings 24–27.
If RTDR or data pump errors occur, check:
Endpoint URLs in Advanced Settings
Operator number in Advanced Setting 12
Public DNS record for endpoint
Roles
Biller
: The organization upgrading from enQuesta v4 to v6.
Project Manager (InvoiceCloud)
: Oversees the upgrade process and coordinates with S&S.
Provider (S&S)
: Supplies integration endpoints and performs backend configuration.
Procedure
Project Kickoff & Configuration Request
Who:
 Project Manager
Contact S&S to request PM and technical contacts.
Request the following items:
Payment codes (tender types)
CIS Service and PayConnect Service endpoints
Inform S&S that test portal setup takes 30+ days and must be initiated ASAP.
Test Portal Setup
Who:
 Project Manager
Submit ITG for test portal configuration once endpoints are received.
Be aware of potential missing custom code (e.g., SEMCO, Atlanta).
Monitor for slow iFrame loading in test (normal behavior).
Advanced Settings Configuration
Who:
 Project Manager
For multi-user billers:
Configure Advanced Settings 24–27 for PayConnect.
Enable Option 127 to sync registered accounts.
For RTDR API compatibility:
Set Advanced Setting 131 to:
= {SystemsAndSoftware": {"RTDRRequest": "ACCOUNTSUMMARIESBYACCOUNTIDS"}}
Maintenance Window & Conversion Weekend
Who:
 Project Manager
Confirm and schedule maintenance window to begin at batch close on Thursday.
Verify that S&S has created a public DNS record for all endpoint URLs.
IC IT makes configuration changes and runs manual test on Friday.
S&S performs their changes Friday or Saturday.
Conversion team supports biller testing on Saturday or Sunday.
Run a manual test to confirm RTDR and data pump endpoints are functional.
Post-Go Live Validation
Who:
 Project Manager
Confirm that Token1/Token2 are being returned correctly to S&S.
If not, re-check Advanced Settings 24–27 and retest.
Monitor for any lag in Capricorn iFrames (expected in test, not in production).
