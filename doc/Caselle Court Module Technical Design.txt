User Interaction and Design
Data Flow - Invoice
Biller will generate invoice file out of Caselle Court Module in the Caselle Native format and Invoice Cloud will convert to BIF. Invoice file will be sent via SFTP to Invoice Cloud. 
Data Flow - Adjustments
Biller will generate adjustment file out of Caselle Court Module in the Caselle Native format and Invoice Cloud will convert to BAF. Adjustment file will be sent via SFTP to Invoice Cloud.
Data Flow - Payment
Invoice Cloud will generate payment file in the Caselle Native format. Payment file will be sent via SFTP after batch close and will only include approved payments.
Data Flow - Void
Voids are done manually via Invoice Cloud Biller Portal.
Data Flow - ACH Reject
ACH Rejects need to be manually adjusted in Caselle.
Payer Experience
Payers will access Invoice Cloud from the Biller website (URL redirect). Payers will use the IC Virtual Site and Customer Portal.
Biller Experience
Standard Biller Portal experience.
