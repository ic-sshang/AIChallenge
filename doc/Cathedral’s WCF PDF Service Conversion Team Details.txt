Summary
This document outlines the steps for implementing the Cathedral WCF service to retrieve invoice PDFs for use with InvoiceCloud. It includes setup instructions for the sample application, integration into existing .NET environments, and usage of service methods for retrieving invoice URLs by account or statement number. The document also includes client-specific reference data and error-handling guidance to ensure a successful implementation.
 Process
Definitions
Account Number: Unique identifier for a customer’s billing account.
Archive Name: The archive location used in the WCF service to retrieve PDFs.
Bill Type: Type of bill (e.g., RE for Real Estate, PP for Personal Property).
Primac: Unique identifier used by Cathedral to distinguish clients.
Report Name: The report template used to generate the PDF.
Statement Number: Unique identifier for a billing statement.
WCF (Windows Communication Foundation): A Microsoft framework for building service-oriented applications.

Quality Assurance
Project Manager
:
Confirm that invoice URLs are correctly retrieved and rendered.
Validate that error messages are handled gracefully.
Ensure HTTPS is used for all PDF requests.
Biller
:
Verify that correct PDFs are returned for each account or statement.
Confirm that URLs expire as expected and are not reused.
