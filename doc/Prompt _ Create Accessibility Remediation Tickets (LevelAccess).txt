Finalized Output Example:
Prompt
Based on the ACCESSIBILITY REQUEST and the BACKGROUND information, please provide the following in the specified format (see EXAMPLE).

For the User Story:

There only needs to be 1 User Story, it should summarize all Findings included from LevelAccess concisely while highlighting the main improvements for each Finding. Identify the specific .aspx filename(s) the Findings mention and explicitly mention them as `filename` in the User Story.

For Acceptance Criteria:

Each Finding from LevelAccess should have its own Acceptance Criteria. You should refer to specific elements the Findings reference. Each Acceptance Criteria should have a title that lists the Finding Number and Crit Number. Each Finding must include a link titled 'Link to Finding' that is extracted from the Accessibility Request data as the first item under its title.

For scenarios:

Write them from the perspective of the user using the system, such that they can be tested by QA.  Each Finding from LevelAccess should have its own Scenario.

EXAMPLE-BEGIN

## User Story

As a user, I should expect [desired accessibility improvements, behavior or functionality], so that [reason it's beneficial, compliant with accessibility standards].

## Acceptance Criteria

### Finding 1 - [short description]

- [Link to Finding]
- [define criterion]

### Finding 2- [short description]

- [Link to Finding]
- [define criterion]
- etc.

## Scenarios

### Scenario 1 - [short description]

Then [outcome].

### Scenario 2 - [short description]

- etc.

EXAMPLE-END
