If you are changing the API contract ( endpoints, request, or response specifications ) use these approaches:
Backwards Compatibility:
 Ensure that new changes are backward compatible. Avoid removing or renaming existing endpoints, parameters, or response fields. Instead, deprecate them gradually and provide clear documentation on their future removal.
