Background
This process is meant to address instances of our system not able to process records in a RPPS file due to issues such as BillerID matching errors. See DS-5251 for an error example.
File Locations
Fiserv

MCRPPS

Fiserv
Find the File
Based on the date the issue occurred you should be able to find the correct file. For example if the date in questions is 12/13/2023, the file name should be called
prod.invoicecloud.remit.20231213005125.processed
 . 
Identify the records
To identify the records you will need the BillerId. Search the document for the BillerId and if there is a record you should be able to find the data line for the Biller in questions. 
Example for BID 3965
Create New File
Using the file in questions, follow the steps below to create the new RPPS file
File Location
The file should be created in the designated area for Fiserv. See File Locations section.
File Name
The filename should be name based on the date is intended to run, thus if today is 12/13/23 then the file  should be named “prod.invoicecloud.remit.202312
14
005125
1111” 
since it would be expected to run on 12/13/23. Also please notice that ones have been added towards the end of the file, this is the standard practice to identify that the file is a rerun, and to prevent issues with other incoming files.
File content:
First Line should be the File header which is located at the first line of the original RPPS file.
Following line should be the Biller’s header
Followed by the Records
Followed by the Biller’s footer
Then after any other Biller using their Biller Header, Records, and then after Footer
Lastly, the File Footer which is located at the last line of the original RPPS file.
