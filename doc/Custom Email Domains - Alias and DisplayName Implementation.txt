Overview:
 
To support sending email via configured email alias or display name per Invoice type when email is sent from a custom email domain else the default alias and display name will be used. 
 
Technical Details:
 
Database Changes: 
Table: BillerEmailDomainInvoiceTypes 
Add two new columns 
FromEmailAlias
, 
FromEmailDisplayName.
 
New procedure to update these 2 columns based on domainid and invoiceTypeId combination. 
 
e.g. 
updBillerEmailDomainInvoiceTypes.
 
Update 
insBillerEmailDomainInvoiceTypes, 
selBillerEmailDomains
 
to support these 2 new columns as inputs. 
 
UI + Backend Changes: 
Updated BillerEmailDomains.aspx and BillerEmailDomains.aspx.vb with logic to support bulk assigning alias to multiple invoice types. 
A textbox to take ‘Email Alias’ or 'Display Name' as input from user and buttons Assign Alias and Assign Display Name is provided. When Assign is clicked, the alias/displayname will undergo few validations and show appropriate error message in the UI when validation fails. 
If validation passes, the input alias/displayname will be then assigned to all selected invoice types at once and the table in UI will be updated. 
This configured alias/displayname will then be used for sending email for that specific invoice type.
