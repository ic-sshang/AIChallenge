A consumer is an entity that can be authenticated and authorized to use the APIs managed by Kong.
Biller consumers must be created under the Biller workspace for the target environment.  Workspaces isolate consumers, services, and routes.
Create consumer
Username
The username must be unique and is used to easily identify the consumer type and primary identifier.
Format
biller-{BILLER_ID}
Example
biller-2002
Custom ID
The custom identifier field will be used to hold meta data about the biller.  This field value is sent to backends as header 
X-Consumer-Custom-Id
.  The value contains key value pairs separated by commas 
,
 and delimited by colon 
:
apikey
 Is used to map the authenticated consumer to the legacy BASIC authentication implementation in the backend APIs.   A Request Transformer Advanced plugin is used to parse the consumer custom id header and replace any existing Authentication header with the legacy BASIC authentication. 
Format
guid:{BILLER_GUID},id:{BILLER_ID},apikey:{BILLER_LEGACY_APIKEY}
