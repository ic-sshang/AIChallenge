SOAP API Information
API Collection
: 
CloudIVR WSDL
Current API Request Endpoint
: 
AdvancedIVRSearch
Proposed New SOAP Response Parameters
CustomerGUID
: Unique identifier for the customer.
AccountNumber
: User's utility account number.
OfferPaperless
: Indicates if paperless billing is available from the biller.
OfferAutopay
: Indicates if autopay is offered by the biller.
EmailOnAccount
: Specifies if an email address is available on the account.
EnrolledInPaperless
: Shows the paperless billing enrollment status.
EnrolledInAutopay
: Shows the autopay enrollment status.
AllowAutopayFromCreditCardDebitCard 
- The Biller Allows the Autopay through Credit Card and Debit Card or not

Analysis for Adding Parameters
Source of Parameters:
CustomerGUID, AccountNumber, EnrolledInPaperless, EnrolledInAutopay, EmailAddress
:
Found in the 
[dbo].[Customers]
 table.
Accessible through the existing stored procedure: 
[dbo].[selCustomers]
.
Implementation:
IVRTechV1.vb - Method
: 
SetAdvancedSearchReturnValues
Code to populate the parameters: 
CustomerGUID 
 AccountNumber 
 EmailOnAccount 
EnrolledInAutopay 
 EnrolledInPaperless 
RETVAL.CustomerGUID = myCustomer.CustomerGUID
RETVAL.AccountNumber = myCustomer.AccountNumber
RETVAL.EmailOnAccount = Not String.IsNullOrWhiteSpace(myCustomer.EmailAddress)
RETVAL.EnrolledInAutopay = Convert.ToBoolean(myCustomer.AutoPay)
RETVAL.EnrolledInPaperless = Convert.ToBoolean(myCustomer.PaperInvoices)
OfferPaperless, OfferAutoPay
Confugured via CRM for specific Invoice Type
Found in the 
[dbo].[billerinvoicetypesettings]
 table.
Accessible through the existing stored procedure: 
[dbo].[selBillerInvoiceTypeSettings] 
.
SP call to get the parameters:
exec [dbo].[selBillerInvoiceTypeSettings] @BillerID = 2689, @InvoiceTypeID = 42, @InvoiceTypeSettingID = '1' -- or '4'
AllowAutopayFromCreditCardDebitCard
Can be configured from CRM under  InvoiceTypes > Invoice Type > AutoPay Configuration.
Found in 
[dbo].[AutoPaymentConfig]
 table.
Accessible through the existing stored procedure: 
[dbo].[selAutoPaymentConfig] 
.
SP call to get the parameters:
exec [dbo].[selAutoPaymentConfig] @BillerID=2689, @InvoiceTypeID = 42
