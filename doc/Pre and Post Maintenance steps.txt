Pre maintenance window activities
Previous night
Kick off Maintenance pipeline and make sure it runs through first step to reschedule applicable jobs to run outside maintenance
On the morning of Maintenance:
Run Starting maintenance step on pipeline run. This will stop the skybot scheduler and must complete before maintenance work starts
[Integrations] Disable the Datapump Controller Not Running alert condition in new relic:
https://onenr.io/0nQxWBGzAjV
[Integrations] (If web traffic is stopped) Stop the ICFTPExportProcessor Azure function and take note of the time it was disabled.
Verify that Pingdom in maintenance mode
Notify DailyOps channel of start of maintenance
Putting up the maintenance window
Login to Cloudflare, go to the 
Invoicecloud.com
 site
Go to DNS
Click on Records
Go to the CNAME record for www and change where it points to from pal.invoicecloud.com to icmaintenance.azurewebsites.net
Go to the CNAME record for my and change where it points to from pal.invoicecloud.com to icmaintenance.azurewebsites.net 
The maintenance plan site is hosted on App Services/Web App in Azure and can be found 
here
Post maintenance
Kick off Finishing maintenance step on pipeline run to bring up skybot scheduler
Once scheduler is back up, rerun any applicable jobs (Batch closes, FTPExports, etc) on Skybot/etc.
[Integrations] (If web traffic was stopped) Run all the missed export windows from ICFTPExportProcessor.
Once work is complete run Post Maintenance - Integrations step on pipeline
[Integrations] Enable the Datapump Controller Not Running alert condition in new relic.
After 12PM run Post Maintenance - Product Engineering step on pipeline
Notify DailyOps channel of finish of maintenance with a synopsis of what was done
Taking down the maintenance window
Login to Cloudflare, go to the invoicecloud.com  site
Go to DNS
Click on Records
Go to the CNAME record for www and change where it points back to pal.invoicecloud.com
Go to the CNAME record for my and change where it points back to pal.invoicecloud.com
