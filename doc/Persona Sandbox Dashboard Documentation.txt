Persona Sandbox Dashboard Documentation
The 
Persona Sandbox
 provides a safe environment to test 
KYC, KYB, and AML flows
 before deploying them in production. It mirrors the production dashboard, but all data is 
test-only
.
The Sandbox dashboard is organized into 
four main sections
:
Identity
Developer
Admin
Platform
Each section has menus and submenus designed for different use cases.

, where you simulate and validate verification flows.

Submenus / Components
Purpose / Use Case
Verifications
Gov ID, Selfie, Document, Database lookups
Inspect individual verification attempts. View pass/fail results, metadata, extraction quality, and fraud signals.
Transactions
Transaction Types, Analytics
Group checks together (e.g., KYC, KYB, AML). Define Transaction Types with required API fields. Analytics help track transaction volume and success/failure rates.
Reports
Watchlist, Adverse Media, Business Registrations, Associated Persons
Generate compliance-ready summaries of test transactions. Useful for AML and due diligence simulations.
Cases
Queues, Manual Review UI, Attachments, Assignment Policies
Test manual review workflows. Developers can simulate case escalation, assignment, and resolution.
Accounts
Account Listings, Types, Field Schemas
Sandbox profiles for individuals/businesses. Developers can test account linking, reference IDs, and audit views.
Inquiries
Inquiry Templates, Sessions
Track identity check flows at the session level. Useful for debugging how users progress through sandbox inquiries.
