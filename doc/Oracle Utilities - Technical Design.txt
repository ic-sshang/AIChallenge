4.* Third party automatic payment extract file (TAPEX)
Description
A TAPEX file will be uploaded to IC via SFTP. The TAPEX file contains autopayments scheduled on the Oracle side for that given day. It needs to be converted into an 
Invoice Update File
 for payment processing. TAPEX files will be generated nightly and need to be processed prior to IC processing autopay so that the 
Custom1:autoPayStagingId
 is available.
IC Use
DataPump - autopay
Partner Batch Process Name
C1-TAPEX
TAPEX File Specs
Each record contains the following comma-separated values, in order:
accountId,autoPayAmount,dueDate,maximumWithdrawalAmount,autoPayStagingId
The footer contains the total number of records in the file and the total amount to be collected, as such:
numberOfRecordsProcessed,totalAmount
Note: 
autoPayStagingId
 needs to be stored in a custom field for confirming/canceling payments in the future
IUF Conversion Specs

 → 
accountId
InvoiceNumber
 → required in IUF, look up by 
AccountNumber
 and 
InvoiceDueDate
if no invoice matches on 
AccountNumber
 and 
InvoiceDueDate
, use the oldest non-zero invoice
InvoiceDueDate
 → 
dueDate
AutoPayCollectionDate
 → always set to the current date in format 
yyyy-MM-dd"T"hh:mm:ss[+|-]hh:mm
Custom1
 → 
autoPayStagingId
