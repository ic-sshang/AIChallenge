Library Name:
 IcReactWebify
Purpose
Converts React components into standard Web Components (Custom Elements) usable in plain HTML or any frontend framework.
Overview
IcReactWebify
 is a lightweight library that bridges modern React components with the Web Components standard. It allows React-based design systems or widgets to be embedded in any frontend environment—vanilla JS, Angular, Vue, or legacy applications—without directly bundling React.
The library supports both Shadow DOM and Light DOM rendering, automatic attribute-to-prop transformation, and React event forwarding to native DOM events.

How to Use
Define Your React Component
typescript
// HelloWorld.tsx
import React from 'react';

interface HelloWorldProps {
  name: string;
}

const HelloWorld: React.FC<HelloWorldProps> = ({ name }) => {
  return <div>Hello, {name}!</div>;
};

export default HelloWorld;

Register the Web Component
typescript
tsx
import React from 'react';
import * as ReactDOMClient from 'react-dom/client';
import HelloWorld from './HelloWorld';
import { IcReactWebify } from 'icreactwebify';

IcReactWebify('hello-world', HelloWorld, {
  React: React,
  ReactDOM: ReactDOMClient,
  shadow: 'open',
  props: { name: 'string' }
});

You can now use 
<hello-world name="John"></hello-world>
 in any HTML page.

Use Light DOM to allow global styles to apply directly.
Limitations
Props must be serializable as string attributes. Use 
'json'
 for objects or arrays.
Event forwarding must be explicitly defined via 
events
.
Dynamic slotting, portals, or React context sharing across the DOM boundary are not supported.
Summary
IcReactWebify
 offers a clean, modular, and framework-agnostic way to transform your React components into embeddable web components. Ideal for:
Micro-frontend architectures
Design systems shared across apps
Embeddable widgets and SDKs
Legacy and cross-framework applications
