Introduction
The 
Persona API
 provides server endpoints for integrating 
identity verification, compliance, and fraud prevention
 into your applications.
With server APIs, you build the 
collection flow
 for users, and pass collected data directly to Persona. This approach requires the 
largest engineering effort
 but provides 
full backend control
.

Idempotence
Safe retries with:

Works only with

Keys expire after 24 hours.
Pagination
Cursor-based pagination:
page[after]
, 
page[before]
, 
page[size]
.
Default size = 10, max = 100.
Serialization
Responses follow 
JSON:API spec
:
data
 ‚Üí main resource
included
 ‚Üí related resources
meta
 ‚Üí metadata
Rate Limiting
Environment limit:
 300 requests/min.
Product quotas:
 e.g., 150 Inquiries/min.
Over limit ‚Üí 
429 Too Many Requests
.
Downloading Files
Files (ID photos, selfies) served via:
https://files.withpersona.com/[filename].[ext]?access_token=[token]

Valid 6h by default (up to 3d configurable).
Error Handling
4xx
 ‚Üí Fix request (e.g., 400 invalid params, 401 invalid key, 409 conflict, 429 too many requests).
5xx
 ‚Üí Retry with exponential backoff.
Network errors
 ‚Üí Retry with same idempotency key.
