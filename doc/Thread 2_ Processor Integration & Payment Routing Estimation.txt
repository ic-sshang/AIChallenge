{To be revisited after a deep-dive discussion with other stakeholders}
Define a universal transaction ID (MO#) to enable routing, reporting, and reconciliation across Chase and non-Chase rails. This is the anchor key for Finance, Ops, and Biller reconciliation.
Note: Why MO# Matters So Much
Single source of truth
: It is the only 
universal transaction ID
 across Chase Commerce, non-Chase rails, IC ledger, and Finance Ops.
Primary join key
: All downstream reconciliation (settlement ingestion, exception management, Finance dashboards) is done on MO#.
Hard Chase limit
: Only one MO# can be echoed back per txn, 
max 40 chars. Api supports 40chars but reports 22 chars.
That‚Äôs a constraint we must design around.
