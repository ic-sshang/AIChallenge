Reports and Fields
This documentation provides an overview of the reports and fields you can create using the GraphQL API. The schema includes various types, connections, and filters to facilitate querying and managing data related to ACH rejects, customers, invoices, and payments.
Reports
ACH Rejects Report
The ACH Rejects report provides detailed information about ACH reject transactions. You can query the following fields:
payment
: The payment associated with the ACH reject.
achRejectTypeID
: The type identifier for the ACH reject.
rejectDate
: The date when the ACH reject occurred.
originatorID
: The identifier of the originator of the ACH transaction.
customerName
: The name of the customer associated with the ACH reject.
tranType
: The type of transaction for the ACH reject.
recDfiRte
: The receiving depository financial institution routing number.
recDfiDda
: The receiving depository financial institution demand deposit account number.
rejectAmount
: The amount of the ACH reject.
customerNumber
: The customer number associated with the ACH reject.
addendaType
: The type of addenda record for the ACH reject.
rejectCode
: The code indicating the reason for the ACH reject.
origTrace
: The original trace number for the ACH transaction.
correctionInfo
: The correction information for the ACH reject.
wsReportID
: The identifier of the WS report associated with the ACH reject.
rejectReference
: The reference for the ACH reject.
adjustmentProcessed
: Indicates whether the adjustment for the ACH reject has been processed.
dateRecAdded
: The date when the ACH reject record was added.

: The UTC date when the ACH reject record was added.

: The date when post-processing for the ACH reject was completed.

: The date when the ACH reject was returned.

: The date when the RMM for the ACH reject was processed.

Example Filter Invoices by Due Date and Sort by Total Amount Due
This query retrieves the first ten invoices filtered by a due date range and sorted by the total amount due in descending order.
graphql
{
  invoices(
    first: 10
    where: {
      invoiceDueDate: {
        gte: "2023-01-01T00:00:00Z"
        lte: "2023-12-31T23:59:59Z"
      }
    }
    order: {
      totalAmountDue: DESC
    }
  ) {
    nodes {
      invoiceNumber
      invoiceDate
      invoiceDueDate
      totalAmountDue
      totalTaxAmount
      totalDiscountAmount
      totalShippingAmount
      vatTaxRate
      vatTaxAmount
      minimumAmountDue
      terms
      balanceDue
      initialBalanceDue
      poNumber
      salesRep
      shipDate
      shipProvider
      trackingNumber
      customerShipName
      customerShipAddress1
      customerShipCity
      customerShipState
      customerShipZip
      customerShipPhone
      convenienceFee
      creditCardServiceFee
      achServiceFee
      lateFeeAmount
      lateFeeDate
      lateFeeDesc
      emailLateFeeNotice
      balanceForward
      lateFeeApplied
      promptPayDiscountAmount
      promptPayExpirationDate
      promptPayDesc
      paymentExpirationDate
      paymentExpirationMessage
      pdfFileName
      pdfToken
      dateRecAdded
    }
  }
}
Pagination
The GraphQL API supports the connection model and provides a standard mechanism for slicing and paginating a result set.
json
{
  invoices(first: 10, after: "opaqueCursor") {
    nodes {
      invoiceNumber
    }
    pageInfo {
      hasNextPage
    }
  }
}
