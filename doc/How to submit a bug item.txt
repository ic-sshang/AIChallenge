Purpose
This document is a guide on how to create, format, and submit a bug item . 
Scrum Assignment
Bug items are submitted directly to the scrum team’s project. If you are not certain, assigned it to the scrum team who last worked on the code and bring it up to discussion to 
Bug Court
. For integration’s scrum teams, they will continue using the ITG board. Please see 
How to submit an Integrations Engineering Service Request (AKA Modification)
 .
Process
Click on the Create button to create the issue.
Before you continue creating your bug item, please be sure that all fields are showed.  
Mandatory Fields
You must include information in the following fields:
Summary
Priority (See 
Priority Assessment
)
Description (See 
https://invoicecloud.atlassian.net/wiki/spaces/DS/pages/2453962755/How+to+Create+A+Prod.+Engineering+Bug+Item#Bug-Template
 
Linked Work Items (link to you DS or SR item)
Incident Number (If relating to an incident)
Bug Template 
Please follow the this format in the summary and description section. Replace the instructions the brackets with your specific information this will help keep all bug reports uniform.
Summary (Title of Jira Item):
 
[Name of Affected Service] - [Short description of Bug] 
Description (Body of Jira Item): 
Fault: 
[Clear and concise description of the issue]
Context
: [Expand on the fault by adding more context regarding the issue.]
How was the bug exposed to the end user?:
 [Provide a summary explanation of how the end user encountered the issue. 
DO NOT
 just state “While researching DS-1234.“]
Expected Result: 
[Experience that the user expects]
Actual Result: 
[Experience that is actually happening]
Steps To Reproduce: 
[Step-by-step guide on how to recreate the bug]
Visual Proof:
 [Image or video of the bug and/or its symptoms]
Impact: 
[Description of 
billers 
and 
software partners
 that could be affected]
Initially Reported By: 
[Biller/Software Partner who reported the issue]
Bug Start Date
: [When was the issue introduced]
Bug Source: 
[Provide the JIRA items that Introduced the bug. Accomplish this by using blame in the repo to see when the changes were introduced]
Source Code:
Repo:﻿ [Code Repository where the bug is located]
Page(s): [Page where the bug is located]
Method(s): [Method where the bug is located]
Line(s): [Line of code where the bug is located]
Priority
 Assessment 
Use the 
Defect/Bug Priority Rubric
 to assess and classify the priority of an item.
Submission
Once you update all the mandatory fields, click the “Create” button. After you submit the items you, or a proxy must be available to attend and present during bug court.
