SRE Ticket has to be created for the team to track their effort. 
Normal monitoring of the job involves
Checking for the status - Manual - in the future this will be info or dashboard driven
Incident Identification - Manual - in the future this will be alert driven
Ticket Creation - When something doesn’t work as expected
Mitigation Action 
Document the finding in the SRE ticket and close it
File an SR ticket to either the eSRE or SRE Observability team for telemetry or alerting gaps
For the eSRE team - make sure the work item is set to Observability and add the label ESRE
Submit a Site Reliability (SR) Item for ESRE
 
For the SRE Observability/ SRE Altimetrik team - make sure the work item is set to Observability and add the label “SRE Altk”
In case you find there is a scope for improving Serviceability - make sure the work item is set to Serviceability
Submit a Site Reliability (SR) Item for Initiatives
