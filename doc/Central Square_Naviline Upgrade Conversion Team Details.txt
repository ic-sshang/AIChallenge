Summary
This document outlines the process for managing a platform upgrade involving AMS, Sungard, or Central Square systems. The upgrade typically requires minimal technical changes but does involve coordination to ensure payment processing is paused appropriately and that no disruptions occur during the maintenance window.
 Process
Definitions
AMS / Sungard / Central Square
: Legacy and current CIS platforms used by municipalities and utilities.
Fusion API
: Integration layer used to connect Central Square systems with external services.
Maintenance Window
: A scheduled period during which system changes are made and payment processing is paused.
Key Responsibilities
Biller
:
Confirm need to pause payments to Central Square during upgrade.
Notify internal teams of the maintenance window.
Project Manager (InvoiceCloud)
:
Confirm and schedule the maintenance window.
Verify that no changes are needed to files or Fusion API.
Coordinate with internal teams and the biller.
Quality Assurance
Project Manager
:
Confirm that payment flow resumes correctly post-upgrade.
Validate that no file or API changes occurred during the upgrade.
Roles
Biller
: The organization undergoing the AMS/Sungard/Central Square upgrade.
Project Manager (InvoiceCloud)
: Oversees the upgrade process and ensures technical continuity.
Procedure
Upgrade Planning
Who:
 Project Manager
Confirm upgrade go-live date with biller.
Verify whether any changes are expected to files or Fusion API.
Maintenance Window Coordination
Who:
 Project Manager
Schedule a maintenance window in CRM.
Ensure payments to Central Square are paused during the window.
Confirm messaging with the biller if customer-facing communication is needed.
Post-Upgrade Validation
Who:
 Project Manager
Confirm that payment processing resumes as expected.
Validate that file formats and API integrations remain unchanged.
