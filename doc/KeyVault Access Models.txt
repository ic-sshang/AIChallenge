Legacy model

Managed

Use when:
You need fine-grained control over specific operations.

RBAC (Role-Based Access Control)
Modern model
 integrated with 
Azure-wide RBAC
.
Permissions are granted via 
Azure roles
 (e.g., Key Vault Reader, Key Vault Secrets User).
Managed 
outside the Key Vault
, at the 
subscription/resource group/resource level

Supports

Easier to manage at scale, especially in large organizations.
Use when:
You want centralized access management across Azure resources.
You're using Azure AD groups or want to leverage role inheritance.
You need to manage access at scale or automate via infrastructure-as-code.
