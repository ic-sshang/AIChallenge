Clone the following 
Repo
Create a branch to track the updates for review
git checkout -b update/${ticket_number}/${friendly-comma-separated-message}
Navigate to the “workspace” directory
environments directory
${dr_region}-${location}-${environment}
Example: pri-eus-npd (primary region, east us, non production)
Locate the 
workspaces
 variable map
avd-${username} = {     
vm_name   = "avd-${username}"     
vm_size   = "Standard_D4s_v5"     
disk_size = 500   }
Ensure the username is 15 characters or less, there is validation on the variable and it will fail the terraform apply
Save the change
Run the following commands
git add --all
git commit -m "added ${username} avd
git push
Navigate to the the following 
Pipeline
Click “Run Pipeline”
Select Branch that was created and pushed in previous steps
Set deployment version, release candidate version and hotfix version accordingly
Look at previous runs, if this is a new AVD for a user, up the hotfix by 1 digit for example
Set Environment
[npd]
 for nonprod and 
[prd]
 for production
Set region
[pri]
 or 
[sec]
Set user population
[eus]
 for East US as an example
Click “Run”
Ensure Terraform Plans are reviewed, and approvals are completed
