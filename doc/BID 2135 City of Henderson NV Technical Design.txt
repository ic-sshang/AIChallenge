Notes: Spoke w/Stephanie and Albert on this export - the custom records sent to IC in the BIF file will need to be sent in the payment export.  There will be a one to many relationship between the single payment in Invoice Cloud and rows exported to the payment file. One payment will equal many rows because there are multiple Invoice Detail lines for Invoice Type ID 96, and for Invoice Type IDs 6 and 95. Every payment must contain a complete set of the custom fields send in the BIF File. The biller’s cashiering system will import these custom fields and apply the payment using that data. Invoice Cloud’s payment file exists as a bridge between the billed detail amounts in PeopleSoft and how to apply the payment to the cashiering system - iNovah .  Cashiering must have this data in place for the integration to work.
Data Sorting by Invoice Type:  Payments lines on Invoice Type ID 6 need to be descending by Invoice Line Number (field 84 of Invoice Detail Record). Payments made to Invoice Type ID 96 need to be ascending by Invoice Line Number (field 84 of Invoice Detail Record).
