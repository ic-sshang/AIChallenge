Helpers:
PdfAnalyzer - Uses the appropriate library to get the words from the pdf
SplitPages - BitMic Wrapper to split and save pdf pages.
DynamicSearcher - BitMic Wrapper used to find text and generate markers with the found data.
Pdf Linking Process:
BillerFTPPDFProcessing_Zip - Moving zip files into server to extract contents then move it back to the ftp or to a different server based on the contents.
BillerFTPPDFProcessing_Pdf - Moving pdf files based on the move action to the approprate server location.
    - Custom move action to move files into directories for the BillerPDFRemoteLinkingReferenceProcessing.
    - Custom move action to move pdfs into blob storage.
Pdf Identification - To detemine the invoice type as well as identify if the PDF needs bursting.
    - Uses self-healing via the use of DynamicSearcher
Burst Pdfs - Using SplitPages class.
    - InMultiplesOf
    - AsDynamicPages
Full Extraction ( Deprecated; Only for BID 56 Town of Hingham )
Partial Extraction - To find account and/or invoice number data to add as the name of the PDF
PdfQueueByFilename - copys extracted pdfs to biller blob path (ic-pdf-<bid>) and then creates a new DB row in BillerPendingPdfs
    - DB row contains the matching blob name that the pdf was renamed to.
PDFQueue_MonitorTT - (Delegation process: Timer Triggered - 20 min intervals) Runs MultiShard SP selPendingPDF_Billers against master shard and delegates a queue trigger to run based on QueueNumber/QueueName
    - 1 | pdfqueue-invoicenumber
    - 2 | pdfqueue-accountnumber
    - 3 | pdfqueue-linkpdfs 
PDFQueue_UpdateInvoiceNumberQT: Queue Trigger("pdfqueue-invoicenumber")
    - Runs spProcessPdfQueueByFilenameInvoiceNumberRenameByGroupID to set the InvoiceNumber based on the most recent invoices that contain the CustomerID associated with the AccountNumber
    - Once a new invoice number is set, the pdf.BillerPendingPdfs row associated with the PDF is updated to include a valid invoice number in the FileName and InvoiceNumber columns.
PDFQueue_UpdateAccountNumberQT: Queue Trigger("pdfqueue-accountnumber")
    - Runs spProcessPdfQueueByFilenameAccountNumberRenameByGroupID to set the AccountNumber based on the most recent invoices that contain the CustomerID associated with the InvoiceNumber
    -Once a new account number is set, the pdf.BillerPendingPdfs row associated with the PDF is updated to include a valid account number in the FileName and AccountNumber columns.
PDFQueue_LinkPDFsQT: Queue Trigger("pdfqueue-linkpdfs")
    - Runs spProcessPdfQueueByFilenameByGroupID based on the PDFLinkingModeID set in the biller's dbo.IntgInvoiceSummary table.
Other Processes:
Pdf Vendor Archiving - Backups original, unmodified pdf that came from the ftp server, from the pdf server and adds them into a blob storage to be pulled from the CRM if needed.
ClearPendingPDFHistory - Database service’s Elastic Database Jobs - Daily purge of Pending PDFs older than 20 days
