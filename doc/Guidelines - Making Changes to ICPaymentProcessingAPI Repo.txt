Whatever code you touch, you must write an integration or unit test.
Eventually you would add new logic to legacy code but as long as you are inserting new well-tested code you won't have to unit test that part of the legacy code.
Data Access Layer or any external call methods should have an integration test. Whatever object is responsible for this logic should be injectable and mockable, usually an interface.
Service methods will usually utilize the DAL objects to perform business operations, therefore we should have a unit test for each method with the DAL objects mocked.
All testing should be properly organized, and mimic the structure of our code, either by namespace or folder structure.
