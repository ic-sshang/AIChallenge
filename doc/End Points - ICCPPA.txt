IC Chase Payment Processing API
The table below lists the available endpoints. For environments refer to the table in the next section.
End Point
Methods
Description
/api/v1/sale
POST
Process new payment transactions (Credit Card or ACH)
/api/v1/refund
POST
Process refund requests for existing payments. Includes full and partial refunds.
/api/v1/void
POST
Process void requests for  payments (payment needs to be either authorized for CC or PendingApproval for ACH)
/health
GET	
Comprehensive health status with detailed system information
/health/live
GET	
Kubernetes liveness probe - simple health check for pod restart decisions
/health/ready
GET	
Kubernetes readiness probe - check if pod is ready to receive traffic
/health/startup
GET	
Kubernetes startup probe - verify container has started successfully
/health/version
GET	
Get application version information
/health/builddate
GET	
Get application build date and timestamp
/health/uptime
GET	
Get application uptime statistics
